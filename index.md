## 問題解決とは

### 問題解決の 5 タイプ
**問題解決として以下のすべてを想定せよ。**

- Trouble 対応型
    - 突発的に発生した、大きな不具合
    - 例: クレカ無くした、交通事故で怪我した
    - 解決策の実行には緊急性を伴う
- Problem 解決型
    - 緊急性はないが理想と現実にギャップがある
    - 例: 最近寝付きが悪い、組織に事なかれ主義が蔓延してて活力落ちてる
    - 放置されがち、かつ放置されるほど解決が難しくなっていく
- Potential Risk 回避型
    - Trouble や Problem の種に事前に処置を施すことで未然に防ぐ
    - 例: 人間ドックに行く、毎日 7 時間以上睡眠取る
    - 守りの問題解決
- Improvement Oppotunity 型
    - 現状に甘んじずベターを目指す問題解決
    - 例: マラソン自己ベスト更新に向けた努力、健康のための食生活改善、定型業務や会議運営の改善
    - 攻めの問題解決
- Theme 回答型
    - 投げかけられた命題に応える形の問題解決。自問自答含む
    - 例: パートナー「今日の晩御飯何にする？」、自問自答「次のPCはWinとMacどっちにするべきか？」
    - 漠然としたテーマだったり、綿密な回答が要求されることもある
    - 🐰1-4 のタイプすべてにおいて発生すると書いている

### 仕事もプライベートもすべて問題解決
**仕事もプライベートもあらゆる場面で上記 5 タイプを当てはめよ。**

### うまい問題解決とは
**個人レベルではベターを、全体レベルではコンスタントな勝利を、また変化についていくために問題解決を磨け。s**

- 個別レベルでは、最もベターな解決策をつくれ
    - 🐰絶対的な正解はないので、できるだけベターなものを探せ言うてる
- 全体レベルでは、コンスタントに勝ち星を上げろ
    - 🐰仕事も人生も豊かにするためには、7勝3敗くらいにはコンスタントに勝ちたい言うてる
- 高度化する問題に対応し続ける
    - 🐰ライフステージや時代の変化に伴い問題が複雑化するので、対応し続けるために問題解決を磨き続けるべし言うてる

## 問題解決ルーティン（Problem Solving Routine）

### 概要
- 問題解決の型のこと。手順あるいはプロセスと捉えてもいい
- ツール:
    - 空、傘、雨
    - 解読、創案、評価、選択

### 空、傘、雨
テンプレート1: `:` 以降に書け

```markdown
- 空（状況）:
- 雨（解釈）:
- 傘（行動）:
```

テンプレート2: 括弧内を消して書け
```markdown
- (空:状況)
    - (雨:解釈)
        - (傘:行動)
```

チェックリスト:

```markdown
- 空 → 雨 → 傘の順にすべて埋めているか？
- 空を言えるだけの情報収集ができているか？
- 雨に自分の解釈を入れているか？他人の受け売りじゃねえよな？
- 傘は具体的か？ 5w1H は盛り込まれているか？
```

バリエーションのトリガーリスト:

```markdown
- 空は複数出せるか？
- ある空から複数の雨を出せるか？
- ある雨から複数の傘を出せるか？
- 段取りの空雨傘と意味合いの空雨傘を行き来できているか？
    - 仕事の段取りを考えるための空雨傘
    - 仕事の意味合い（意図と期待されたアウトプット）を考えるための空雨傘
```

それ以外のトリガーリスト:

```
- ある空を出したとして、「それ以外」があるよな？
- ある雨を出したとして、「それ以外」があるよな？
- ある傘を出したとして、「それ以外」があるよな？

===

例: 雪が少し積もっていて滑るせいで徒歩 15 分のいつもの定食屋に行けないかもしれない

- 雪が少し積もっていて滑る
    - いつもの二倍の時間をかけて歩くべきだ
        - 早めに出るためのリマインダーを仕掛けよう
        - それ以外
    - それ以外
- それ以外

↓

- 雪が少し積もっていて滑る
    - いつもの二倍の時間をかけて歩くべきだ
        - 早めに出るためのリマインダーを仕掛けよう
        - ★客の具合を知るために店主に電話しよう
    - ★押入れから長靴を引っ張り出すべきだ
- ★雪は少し積もっているが別に滑りはしない（実際に検証する必要アリ）
```
